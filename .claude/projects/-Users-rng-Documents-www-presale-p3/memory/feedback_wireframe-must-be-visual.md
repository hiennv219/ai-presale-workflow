---
name: wireframe-must-be-visual
description: Wireframe sections in proposals must contain visual diagrams (ASCII art or image links), never just text bullet descriptions
metadata:
  type: feedback
---

Wireframe sections must always include visual layout representation — ASCII art at minimum, or image/link placeholders for external tools (Figma, Excalidraw).

**Why:** A "wireframe" described only in bullet-point text is not a wireframe — it's just a feature list. Even at high-level, wireframes communicate spatial layout, hierarchy, and interaction patterns that text cannot convey.

**How to apply:** When producing any wireframe section in a proposal, always render ASCII box-drawing diagrams showing screen layout, field placement, and visual hierarchy. If the deliverable warrants higher fidelity, use placeholder links to external design tools instead of pretending text descriptions are wireframes.
